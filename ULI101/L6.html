Week 2 Materials: Basic Linux File system command


            - More importantly, what kind of text editor should linux system used?
            vi or vim, nano

            - How to use vi or vim?
            vim a.txt //once this command is enter, you will enter into a vi control platform
            i         //press i to get into insert mode
            Esc       //press keyboard Esc to cancel and go back to vi control platform
            :x        //save and exit vim editor

            - How to use nano?
            nano a.txt //once this command is enter, you will enter into nano control platform, you are already into the insert mode
            abc        //type anything you want, in this example typed "abc"
            Ctrl+x     //press keyboard fast key "Ctrl + x " to save the document
            y          //press y (yes) because you want to save the modify File
            Enter      //press Enter because you want the original file name (a.txt), unless you want to save another name

Week 3 Materials: File Location Path


            - Absolute path and Relative to home path does not care where you are (pwd), it will move to the exact address that you have typed.
            - Absolute path:
            /home/yourID/ABC
            
            - Relative to home path:
            ~/ABC

            
            - Relative path names use it usually, but you need to know where you are first (pwd)
            - Relative path (current directory):
            ./
            .

            - Relative path (parent directory):
            ../ABC
            ../
            ..

            - Relative path (move down to one directory):
            ./ABC
            ABC
