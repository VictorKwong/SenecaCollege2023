File Path Name

1. Absolute pathname

Always start with symbol /

It will sent you to the exact location, no matter where you are


            ExampleðŸ”°:
            
            cd /home/userid/week3/notes

            - home directory -> user id directory -> week3 directory -> notes directory (change directory, goes to notes directory)

            cd /home/userid/ABC

            - home directory -> user id directory -> ABC directory (change directory goes to ABC directory)

2. Relative-To-Home pathname

Always start with symbol ~/

~ symbol represent as /home/userid

It will sent you to the exact location, no matter where you are (same principle as Absolute pathname)


            ExampleðŸ”°:
            
            cd ~/week3/notes

            - home directory -> user id directory -> week3 directory -> notes directory (change directory, goes to notes directory)

            cd ~/ABC

            - home directory -> user id directory -> ABC directory (change directory goes to ABC directory)

3. Relative pathname

can start with 5 different cases . ./ .. ../ filename

3.1 "." symbol means current directory, where ./ can type filename afterward, Ex. cd ./ABC

3.2 ".." symbol means move back to parent directory, where ../ can type filename afterward, Ex. cd ../ABC

3.3 "ABC" typing filename, means move down into a directory called ABC

You need to know where you are before typing Relative pathname (pwd)


            ExampleðŸ”°:
            
            cd .

            - move to current directory (does nothing)

            cd ..

            - move back to parent directory (Before. /home/userid ðŸ‘‰ After /home)

            cd ../ABC

            - move back to parent directory, then move down into a directory called ABC (Before. /home/userid ðŸ‘‰ After /home/ABC)

            cd ABC
            cd ./ABC

            - move down into a directory called ABC (Before. /home ðŸ‘‰ After /home/ABC)

Filename Expansion Symbols: * ? [] [!]


            touch a aac ab abbc abc ac adc aec abb
            - create a bunch of empty files to test out Filename Expansion Symbols

            Example1: ls abb*
            Result Expect1: abbc abb
            - * means 0 or more any characters
            
            Example2: ls aa[a-z]
            Result Expect2: aac 
            - [] means 1 character with that specific type(a-z)
            
            Example3: ls a?
            Result Expect3: ab ac 
            - ? means 1 character with any type

            Example4: ls aa[!0-9]
            Result Expect2: aac 
            - [!] means 1 character NOT with that specific type(not 0-9)
